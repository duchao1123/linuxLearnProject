弱类型语言，解释型语言
概括：将需要执行的命令保存到文本中，按照从上到下的顺序执行。
准确叙述：若干命令 + 脚本格式 + 脚本语法 + 逻辑 = shell 脚本

shell脚本基本写法
脚本第一行：魔法字符#！指定解释器【必写】
#! /bin/bash    表示以下内容使用bash解释器解析

#! /bin/env bash  上面指定路径方式，但是系统不同可能找不到具体解释器，兼容性写法如下，可以先从环境中找
第二行：注释
第三行：脚本内容

交互式变量（用户输入变量）
read [选项] 变量名


选项：
-p： 定义提示用户的信息
-n： 限制变量值长度
-s： 不显示用户输入内容
-t： 输入超时.     eg：-t 3  单位秒，超时前需要按回车

读取文件中的变量(常用)
read [选项] 变量名 < 操作

eg:
vim myconfig.txt
host=‘127.0.0.1’

read -p ’host:’ HOST < myconfig.txt

定义有类型的变量
目的：给变量做一些限制，固定变量类型，比如整型、只读
语法：declare 选项 变量名=变量值

选项：
-i： 整数
-r： 只读
-a： 普通数组
-A： 关联数组
-x： 将变量通过环境导出      declare -x AAA=123 相当于 export AAA=123

变量分类：
本地变量： 当前进程生效
环境变量： 当前进程有效，并且能够被子进程调用
  • env查看当前用户的环境变量
  • set查询当前用户的所有变量（临时变量与环境变量）
  • export 变量名=值
全局变量：全局所有的用户和程序都能调用，并且继承，新建的用户也能调用
相关配置文件

文件
说明
$HOME/.bashrc
当前用户的bash信息，用户登录时读取
$HOME/.bash_profile
当前用户的环境变量，用户登录时读取
$HOME/.bash_logout
当前用户退出当前shell时读取
$HOME/.bash_history
历史
/etc/bashrc
全局bash信息，所有用户都生效
/etc/profile
全局环境变量信息
以上文件修改后，都需要重新source，或. 让其生效；或退出重新登录
用户登录系统读取相关文件顺序：
  1. /etc/profile
  2. $HOME/.bash_profile
  3. $HOME/.bashrc
  4. /etc/bashrc
系统变量：shell本身已经固定好了它的名字和作用（不能修改）
